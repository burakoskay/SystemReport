## El modelo de facturación basado en tokens de GitHub Copilot provoca ira entre desarrolladores

La era dorada de GitHub Copilot de Microsoft parece estar llegando a su fin. GitHub Copilot, un asistente de codificación impulsado por IA, ha cambiado a un modelo de facturación basado en tokens, lo que ha generado críticas entre desarrolladores. El cambio marca una desviación significativa de su estructura de precios anterior. Este nuevo modelo cobra a los desarrolladores en función del número de tokens utilizados, lo que puede acumularse rápidamente, especialmente para usuarios intensivos. Los desarrolladores llaman al nuevo modelo de precios 'una broma'.

El modelo de facturación basado en tokens utiliza un sistema complejo para rastrear y cobrar a los desarrolladores por el número de tokens utilizados. Un token representa una unidad de código procesada por Copilot. Cuanto más código escriben o modifican los desarrolladores con la ayuda de Copilot, más tokens consumen. Este modelo ha generado preocupaciones sobre la accesibilidad y el valor de la herramienta, especialmente para usuarios intensivos.

## El auge de GitHub Copilot

Desde su lanzamiento, Copilot ha ganado un seguimiento significativo entre desarrolladores, con muchos que elogian su capacidad para agilizar tareas de codificación y mejorar la productividad.

## Mecánicas técnicas: Cómo funciona la facturación basada en tokens

El modelo de facturación basado en tokens se basa en un algoritmo complejo que rastrea el número de tokens utilizados por los desarrolladores. Este algoritmo tiene en cuenta la cantidad de código procesado por Copilot, así como el tipo de código que se está escribiendo. Por ejemplo, escribir una sola línea de código puede consumir un token, mientras que escribir una función completa puede consumir varios tokens. Este modelo ha generado preocupaciones sobre la accesibilidad y el valor de la herramienta, especialmente para usuarios intensivos.

## Preocupaciones y controversias

Algunos desarrolladores han planteado preocupaciones sobre Copilot que generan código idéntico al código generado bajo licencias de código abierto que no permiten obras derivadas. Esto podría llevar a que los desarrolladores utilicen código protegido por derechos de autor sin saberlo. GitHub ha respondido a estas preocupaciones, afirmando que está trabajando para abordar el problema y garantizar que las características de generación de código de Copilot sean compatibles con licencias de código abierto.

## 침해 사고 소개
GitHub은 내부 저장소에 대한 무단 접근과 관련된 보안 사고를 조사하고 있습니다. 이번 사고는 GitHub 시스템의 보안 및 사용자에게 미치는 영향에 대한 우려를 제기하고 있습니다.

사고는 Hacker News에 처음 보고되었으며, GitHub이 내부 저장소에 대한 무단 접근을 조사하고 있다는 게시물이 있었습니다. 게시물에는 GitHub 공식 계정의 트윗 링크가 포함되어 사고를 확인했습니다.

## 사고 세부 사항
보고에 따르면 사고는 무단 사용자가 GitHub 내부 저장소에 접근하는 것을 포함했습니다. 사용자는 소수의 시크릿에 접근한 후 추가적인 시스템 접근을 얻기 위해 사용했습니다.

GitHub은 탐지 즉시 경고를 트리거했으며, 취약점을 완화하고 키를 회전하고 생산 시스템이나 데이터에 접근이 없음을 확인했습니다. 또한, 사고에 대한 자세한 정보는 앞으로의 블로그 게시물에서 공유할 것이라고 밝혔습니다.

## 이전 보안 사고와의 맥락
이번 사고는 GitHub이 보안 우려를 겪은 최초의 사례가 아닙니다. 과거에 GitHub은 컨설팅 업무에 사용되는 GitLab 인스턴스의 침해를 포함한 여러 보안 사고를 경험했습니다. 이번 사고는 무단 접근에 대비한 강력한 보안 조치를 보호하는 것의 중요성을 강조합니다.

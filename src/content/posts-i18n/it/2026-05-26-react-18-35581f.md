## Introduzione a React 18

React 18 è ora in beta, portando diversi miglioramenti significativi alla popolare libreria JavaScript. La nuova versione è il risultato di un ampio feedback e iterazione da parte del React 18 Working Group, che ha lavorato a stretto contatto con librerie come Redux, Next.js e React Testing Library per garantire un percorso di aggiornamento agevole per gli sviluppatori.

La versione beta di React 18 è completa di funzionalità, senza ulteriori nuove API previste prima della versione finale. Il lavoro rimanente consiste nella documentazione e miglioramenti della qualità della vita, come messaggi di errore migliori.

## Miglioramenti al rendering lato server

Uno dei principali miglioramenti in React 18 è la nuova architettura Suspense SSR. Questa architettura consente un rendering lato server più efficiente, suddividendo l'app in unità indipendenti più piccole che possono essere renderizzate indipendentemente. Ciò si traduce in caricamenti di pagina più veloci e un'esperienza utente migliorata.

La nuova architettura è opt-in e gli sviluppatori possono utilizzare l'API renderToPipeableStream per approfittarne. Questa API è ancora in sviluppo, ma promette di migliorare significativamente le prestazioni del rendering lato server nelle applicazioni React.

## React Router e parametri di ricerca

In notizie correlate, il team di React Router sta discutendo l'implementazione di parametri di ricerca definiti in modo sicuro con tipi in routes.ts. Questa funzionalità consentirebbe agli sviluppatori di definire uno schema per i parametri di ricerca all'interno di ogni voce di percorso, fornendo sicurezza dei tipi sia per href che per le proprietà del componente.

La discussione si concentra su come implementare questa funzionalità senza introdurre collisioni di nomi tra parametri di percorso e parametri di ricerca. Una soluzione proposta è utilizzare un oggetto searchParams separato da Route.LoaderArgs, che impedirebbe collisioni di nomi e fornirebbe una chiara distinzione tra parametri di percorso e parametri di ricerca.

## Meccanica tecnica

La nuova architettura Suspense SSR in React 18 si basa su un concetto chiamato 'suspense', che consente ai componenti di 'sospensionare' il rendering fino a quando non sono soddisfatte determinate condizioni. Ciò consente un rendering lato server più efficiente, poiché i componenti possono essere renderizzati indipendentemente e in parallelo.

L'API renderToPipeableStream è una parte fondamentale di questa architettura, poiché consente agli sviluppatori di approfittare del nuovo modello di rendering basato su suspense. Questa API fornisce un modo per rendere i componenti a un flusso, che può quindi essere indirizzato al client, consentendo un rendering più efficiente e prestazioni migliorate.

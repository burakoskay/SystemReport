## Introduction à React 18

React 18 est désormais en bêta, apportant plusieurs améliorations significatives à la bibliothèque JavaScript populaire. La nouvelle version est le résultat d'un processus itératif et d'un retour d'expérience approfondi du groupe de travail React 18, qui a travaillé en étroite collaboration avec des bibliothèques telles que Redux, Next.js et React Testing Library pour garantir une mise à niveau fluide pour les développeurs.

La version bêta de React 18 est complète en termes de fonctionnalités, sans nouvelle API supplémentaire attendue avant la sortie finale. Le travail restant consiste en des améliorations de la documentation et de la qualité de vie, telles que de meilleurs messages d'erreur.

## Améliorations de rendu côté serveur

L'une des principales améliorations de React 18 est la nouvelle architecture Suspense SSR. Cette architecture permet un rendu côté serveur plus efficace, en décomposant l'application en unités indépendantes plus petites pouvant être rendues de manière indépendante. Cela se traduit par des temps de chargement de page plus rapides et une expérience utilisateur améliorée.

La nouvelle architecture est facultative et les développeurs peuvent utiliser l'API `renderToPipeableStream` pour en profiter. Cette API est encore en développement, mais elle promet d'améliorer considérablement les performances de rendu côté serveur dans les applications React.

## React Router et paramètres de recherche

Dans une actualité connexe, l'équipe de React Router discute de la mise en œuvre de paramètres de recherche sécurisés en termes de type définis dans `routes.ts`. Cette fonctionnalité permettrait aux développeurs de définir un schéma pour les paramètres de recherche à l'intérieur de chaque entrée de route, fournissant une sécurité de type pour les props `href` et de composant.

La discussion est centrée sur la façon de mettre en œuvre cette fonctionnalité sans introduire de collisions de noms entre les paramètres de chemin et les paramètres de recherche. Une solution proposée consiste à utiliser un objet `searchParams` distinct de `Route.LoaderArgs`, qui empêcherait les collisions de noms et fournirait une distinction claire entre les paramètres de route et les paramètres de recherche.

## Mécaniques techniques

La nouvelle architecture Suspense SSR de React 18 est basée sur un concept appelé 'suspense', qui permet aux composants de 'suspendre' leur rendu jusqu'à ce que certaines conditions soient remplies. Cela permet un rendu côté serveur plus efficace, car les composants peuvent être rendus de manière indépendante et en parallèle.

L'API `renderToPipeableStream` est un élément clé de cette architecture, car elle permet aux développeurs de profiter du modèle de rendu basé sur le suspense. Cette API fournit un moyen de rendre des composants à un flux, qui peut ensuite être canalisé vers le client, permettant un rendu plus efficace et de meilleures performances.

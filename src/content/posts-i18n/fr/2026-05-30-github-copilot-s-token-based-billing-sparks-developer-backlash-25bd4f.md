## La tarification basée sur les jetons de GitHub Copilot suscite la colère des développeurs

L'âge d'or de GitHub Copilot de Microsoft semble toucher à sa fin. GitHub Copilot, un assistant de codage alimenté par l'IA, est passé à un modèle de tarification basé sur les jetons, suscitant les critiques des développeurs. Ce changement marque une rupture significative avec sa structure de tarification précédente. Ce nouveau modèle facture aux développeurs en fonction du nombre de jetons utilisés, ce qui peut s'accumuler rapidement, en particulier pour les utilisateurs intensifs. Les développeurs qualifient le nouveau modèle de tarification de "blague".

Le modèle de tarification basé sur les jetons utilise un système complexe pour suivre et facturer aux développeurs le nombre de jetons utilisés. Un jeton représente une unité de code traitée par Copilot. Plus les développeurs écrivent ou modifient du code avec l'aide de Copilot, plus ils consomment de jetons. Ce modèle a suscité des inquiétudes quant à l'accessibilité et à la valeur de l'outil, en particulier pour les utilisateurs intensifs.

## L'ascension de GitHub Copilot

GitHub Copilot a été lancé en tant que compagnon de programmation par paires alimenté par l'IA pour les développeurs, en tirant le contexte du code sur lequel on travaille pour suggérer des lignes entières ou des fonctions entières. La technologie a été développée en partenariat avec *OpenAI*. Depuis son lancement, Copilot a gagné en popularité auprès des développeurs, beaucoup louant sa capacité à rationaliser les tâches de codage et à améliorer la productivité.

## Mécaniques techniques : comment fonctionne la tarification basée sur les jetons

Le modèle de tarification basé sur les jetons repose sur un algorithme complexe qui suit le nombre de jetons utilisés par les développeurs. Cet algorithme prend en compte la quantité de code traitée par Copilot, ainsi que le type de code écrit. Par exemple, écrire une seule ligne de code peut consommer un jeton, tandis qu'écrire une fonction entière peut consommer plusieurs jetons. Ce modèle a suscité des inquiétudes quant à l'accessibilité et à la valeur de l'outil, en particulier pour les utilisateurs intensifs.

## Préoccupations et controverses

Certains développeurs ont exprimé leurs inquiétudes quant au fait que Copilot génère du code identique à celui généré sous des licences *open-source* qui n'autorisent pas les œuvres dérivées. Cela pourrait conduire à ce que des développeurs ignorants utilisent du code protégé par le droit d'auteur. GitHub a répondu à ces préoccupations, en déclarant qu'il travaille à résoudre le problème et à garantir que les fonctionnalités de génération de code de Copilot sont conformes aux licences *open-source*.

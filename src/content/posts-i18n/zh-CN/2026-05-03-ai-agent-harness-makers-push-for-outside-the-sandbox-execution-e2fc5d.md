## 在沙盒外执行的理由

OmoiOS 和 Broccoli 等 AI 智能代理框架的制作者正在推动在沙盒外执行。这种方法有望提高性能和可靠性。OmoiOS 是一个开源的编排运行时，它使用隔离的沙盒中的并行代理群集将规范转换为 PR。

OmoiOS 系统包括一个处理合并并保持一切正常运行的监管代理。它根据现有的代码库生成规范，构建具有真实依赖关系的任务 DAG。系统在隔离的沙盒中运行代理群集，直到工作完成。

## 具体实现

Broccoli 是一款将 Linear 工单转变为已交付 PR 的 AI 智能代理框架，由 Claude 和 Codex 驱动，在 Google Cloud 上运行。该系统作为两个 Cloud Run 工作负载运行，共享一个 Postgres 数据库。

要部署 Broccoli，用户需要一个 Google Cloud 项目、一个 GitHub 应用以及来自 GitHub、Linear、OpenAI 和 Anthropic 账户的特定机密。部署过程从零开始，解释了值的来源，并指出了出于安全原因仍需要手动执行的步骤。

## 利益和权衡

在沙盒外运行代理提供了多种益处，包括更好的依赖关系感知、并行执行以及任务之间的结构化交接。然而，这种方法也引发了有关安全性和控制权的问题。

OmoiOS 和 Broccoli 系统通过隔离的沙盒和监管代理解决了这些问题。然而，用户仍需要权衡在沙盒外执行的益处和风险。

## 更深入地了解 OmoiOS

OmoiOS 旨在自动将规范转换为 PR 的过程。它使用监管代理来管理过程，确保一切正常运行。该系统旨在处理复杂的任务和依赖关系，使其成为开发人员的强大工具。

OmoiOS 的一个关键功能是能够根据现有的代码库生成规范。这允许开发人员在已有基础上进行构建，而不是从头开始。系统还包括具有真实依赖关系的任务 DAG，有助于确保任务按正确顺序完成。

## Broccoli 的架构

Broccoli 的架构旨在利用 AI 代理的强大功能。该系统使用两个 Cloud Run 工作负载，共享一个 Postgres 数据库，从而可以扩展并处理大量工作负载。

Broccoli 的部署过程旨在简单明了，具有清晰的说明和简单的设置过程。但是，用户仍需要注意安全问题，并确保拥有必要的权限和访问权限。

## GitHub Copilot 的基于 Token 的计费模式引发开发者愤怒

微软的 GitHub Copilot 的黄金时代似乎已经结束。GitHub 的 Copilot，一款由 AI 驱动的编码助手，已经转向基于 Token 的计费模式，引发开发者批评。这一变化标志着其定价结构的重大转变。新模式根据使用的 Token 数量对开发者进行收费，尤其对于重度用户，费用可能迅速增加。开发者称新的定价模式「开玩笑」。

基于 Token 的计费模式使用复杂的系统来跟踪和收取开发者使用的 Token 数量。一个 Token 代表 Copilot 处理的一单位代码。开发者使用 Copilot 辅助编写或修改的代码越多，消耗的 Token 越多。该模式引发了人们对该工具的可用性和价值的担忧，尤其是对于重度用户。

## GitHub Copilot 的崛起

GitHub Copilot 作为一款 AI 配对编程伴侣推出，为开发者提供代码上下文，以建议整行或整个函数。该技术与 OpenAI 合作开发。自推出以来，Copilot 在开发者中获得了显著的关注，许多人赞扬其能够简化编码任务和提高生产力。

## 技术机制：Token-Based 计费工作原理

基于 Token 的计费模式基于复杂的算法，跟踪开发者使用的 Token 数量。该算法考虑了 Copilot 处理的代码量以及编写的代码类型。例如，编写一行代码可能会消耗一个 Token，而编写整个函数可能会消耗多个 Token。该模式引发了人们对该工具的可用性和价值的担忧，尤其是对于重度用户。

## 行业背景：AI 驱动的编码助手崛起

围绕 GitHub Copilot 的争议凸显了在 AI 驱动的编码助手市场中平衡创新与公平定价和使用政策的挑战。竞争公司，如 Kite 和 TabNine，提供类似的服务，但定价模式不同。AI 驱动的编码助手市场规模正在迅速增长，采用曲线显示出显著的增长。根据最近的报告，到 2025 年，市场预计将达到 15 亿美元，AI 驱动的编码助手将成为开发者的必备工具。

## 历史：此前发布和先例监管行动

这并不是第一次有公司因基于 Token 的计费模式而面临强烈反弹。此前类似服务的发布引发了对公平定价和使用政策的担忧。监管机构此前曾采取行动解决这些问题，为 GitHub 等公司树立了先例。例如，2020 年，欧盟的通用数据保护条例（GDPR）为数据保护和使用政策设定了新标准。

## 下游影响：谁受益，谁被挤压

基于 Token 的计费模式引发了人们对谁受益和谁被挤压的疑问。Copilot 的重度用户可能会发现新的定价模式不可持续，而轻度用户可能不会受到太大影响。编码社区的影响将受到密切关注，因为 GitHub 响应开发者的强烈反弹。一些开发者表示担忧，新的定价模式可能会不成比例地影响开源开发者，他们严重依赖 Copilot 的功能。

## 未来展望

围绕 GitHub Copilot 的争议凸显了在 AI 驱动的编码助手市场中平衡创新与公平定价和使用政策的挑战。GitHub 对开发者强烈反弹的回应以及对编码社区的影响将受到密切关注。基于 Token 的计费模式引发了人们对该工具的可用性和价值的疑问。随着市场的不断发展，GitHub 将如何适应不断变化的开发者需求和偏好仍有待观察。

## 关注和争议

一些开发者对 Copilot 生成的代码与开源许可证生成的代码相同表示担忧，而这些许可证不允许派生作品。这可能会导致不知情的开发者使用受版权保护的代码。GitHub 已经回应了这些担忧，表示正在努力解决该问题，并确保 Copilot 的代码生成功能符合开源许可证。

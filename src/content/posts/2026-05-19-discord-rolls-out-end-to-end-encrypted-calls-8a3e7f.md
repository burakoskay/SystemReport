## Encryption Comes to Discord Calls
Discord has made end-to-end encryption available for all voice and video calls. The update ensures that even Discord's servers cannot access call content.

## A Privacy Boost for Users
The move addresses concerns about eavesdropping risks. A previous server-side log incident showed Discord could technically access call metadata. With encryption, users now have a concrete privacy guarantee.

## Technical Details and Industry Impact
End-to-end encryption on Discord means the client generates cryptographic keys for each call. The server merely forwards encrypted media packets. This approach is similar to Signal's model but extended to multi-party calls.

## Regulatory and Competitive Context
The European Union's Digital Services Act and prospective US legislation emphasize end-to-end encryption for large user bases. Discord's implementation could serve as a reference point.

## Technical Mechanics
When a user initiates a call on Discord, the client generates a unique cryptographic key for that call. This key is used to encrypt the media packets, which are then forwarded to the server. The server does not have access to the decryption key, ensuring that only the participants in the call can hear each other. This process is similar to how Signal's end-to-end encryption works, but Discord's implementation is extended to support multi-party calls.

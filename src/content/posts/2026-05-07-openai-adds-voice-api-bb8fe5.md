# OpenAI Expands API with Voice Intelligence
OpenAI added voice intelligence to its API, which the company says is handy for customer-service systems and has applications in education and creator platforms. This feature allows developers to integrate voice capabilities into their applications, potentially transforming how users interact with technology.

## Voice API Targets Multiple Domains
The voice API can be used in customer-service bots, education tools, and creator platforms. OpenAI highlights three target domains: customer-service systems, educational apps, and creator platforms. This expansion into voice intelligence could mark a significant shift in how OpenAI's API is utilized across various industries, particularly in sectors where voice-activated tools are becoming increasingly prevalent. For instance, voice-activated customer support bots could become more common, allowing users to quickly resolve issues without needing to interact with a human representative.

## Trusted Contact Safeguard
OpenAI introduced a feature called Trusted Contact to flag conversations that may indicate self-harm. The company is expanding its efforts to protect ChatGPT users in cases where conversations may turn to self-harm. The safeguard is a new safety layer designed to identify and mitigate potential risks associated with AI-driven chat platforms. This move underscores OpenAI's commitment to user safety and demonstrates an awareness of the potential risks associated with AI-driven chat platforms.

## Chrome Codex Plugin
OpenAI released a Codex plugin for Chrome. The extension injects Codex's code-generation engine into any web page that contains a code editor. The plugin could turn Codex into more than a coding aid, making it a versatile tool for developers and non-developers alike. By integrating Codex with Chrome, OpenAI is making its code-generation capabilities more accessible and user-friendly, potentially appealing to a broader audience.

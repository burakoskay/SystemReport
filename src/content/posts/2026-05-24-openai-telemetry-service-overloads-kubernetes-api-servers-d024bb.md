OpenAI deployed a telemetry service at 3:12 PM PST on Dec 11, 2024. The rollout overloaded Kubernetes API servers in large clusters, collapsing DNS service discovery.

The telemetry configuration caused every node in clusters of thousands to issue expensive API calls. Those calls saturated the Kubernetes API servers, breaking the DNS cache that pods use to locate each other.

## Telemetry Rollout Triggers API Saturation

The new service collected detailed control-plane metrics. In production, each node executed resource-intensive list operations that scaled with cluster size. When thousands of nodes performed those operations simultaneously, the API servers became overwhelmed.

The overload manifested as API server saturation. The API servers stopped functioning properly, and the DNS-based service discovery mechanism failed.

## Control Plane Collapse

OpenAI's pre-deployment checks measured CPU and memory usage on a per-cluster basis but missed API-server load assessment. The telemetry service increased load on API servers through authentication and authorization requests generated by kubelet on each node.

The surge in auth traffic strained the API server while other components kept running. Containers did not restart, but the loss of DNS meant services could no longer resolve each other's names.

## Technical Mechanics

The telemetry service's configuration caused every node in each cluster to execute resource-intensive list operations. This resulted in a large number of API calls being made to the Kubernetes API servers, leading to saturation and eventual failure. To avoid similar incidents in the future, companies can consider implementing more efficient data collection and aggregation strategies, such as pulling metrics from a central aggregator instead of each node issuing API calls.

In addition, implementing rate limiting and throttling mechanisms can help prevent API servers from becoming overwhelmed. By prioritizing API requests and allocating resources more efficiently, companies can reduce the risk of outages and ensure the reliability of their systems.
